$15 в месяц. Это вся моя инфраструктура на 9 продуктов. Вот каждый сервис, его бесплатный лимит и момент, когда он перестанет быть бесплатным.

## Compute: Cloudflare Workers — $0

Cloudflare Workers — это serverless-платформа с генерозным бесплатным тарифом: 100 000 запросов в день, 10 мс CPU на запрос.

**Что делает в моём стеке:** хостинг статических сайтов через Pages, API-прокси (обход блокировок Groq из России), relay для Telegram-вебхуков, newsletter-подписка.

**Когда ломается:** при CPU-intensive задачах. 10 мс на запрос — это мало для тяжёлых вычислений. Для ML-инференса не подходит — для этого есть RunPod.

**Когда придётся платить:** при >100K запросов/день. Workers Paid — $5/мес за 10M запросов. До этого момента далеко.

## Хостинг Next.js: Vercel — $0

InkCloak работает на Next.js и хостится на Vercel. Бесплатный тариф: 100 ГБ bandwidth, serverless functions, автоматический деплой из GitHub.

**Когда ломается:** при >100 ГБ трафика или при необходимости edge middleware. Для моего объёма — достаточно на ближайшие месяцы.

## База данных: Supabase + SQLite — $0

Supabase даёт 500 МБ PostgreSQL бесплатно. Для CoffeeBot хватает с запасом. AICPO использует SQLite (файл на сервере) — ноль стоимости, нулевая задержка.

**Когда ломается:** SQLite не масштабируется горизонтально. При >10 конкурентных записях в секунду нужен PostgreSQL. Но для MVP и первых 1000 пользователей SQLite — идеальный выбор.

## LLM-инференс: Groq — $0

Groq предоставляет бесплатный API для LLM-моделей: Llama 3.3 70B, Qwen3, Mixtral. Лимит: 14K tokens/min, 30 запросов/мин.

**Что делает в моём стеке:** чат в AICPO (основной провайдер), классификация интентов, извлечение фактов, генерация brief. Всё это бесплатно.

**Когда ломается:** при высокой нагрузке. 30 req/min — это примерно 10 одновременных пользователей чата. При масштабировании нужен fallback на OpenRouter (тоже есть бесплатные модели).

**Нюанс для России:** Groq блокирует российские IP. Решение — Cloudflare Worker как прокси. Запрос идёт через CF Worker (нидерландский PoP), Groq видит IP Cloudflare.

## Аналитика: PostHog — $0

1 миллион событий в месяц бесплатно. Сессии, воронки, когорты, A/B-тесты. Альтернатива Mixpanel и Amplitude.

**Когда ломается:** при >1M events. На моём объёме это произойдёт не скоро. Но PostHog умный — можно сэмплировать события и оставаться в лимите.

## Email: Loops.so — $0

1000 контактов бесплатно. Drip-кампании, сегментация, аналитика. Использую для newsletter nevrai.com.

**Когда ломается:** при >1000 подписчиков. Следующий тариф — $49/мес. К тому моменту newsletter должен приносить достаточно value, чтобы оправдать расход.

## Мониторинг: Sentry — $0

5000 ошибок в месяц бесплатно. Отслеживание exception, performance monitoring, source maps.

**Когда ломается:** при >5K ошибок. Если у вас 5000 ошибок в месяц — проблема не в Sentry, а в коде.

## ML-тренировка: RunPod — $0.49 разово

RunPod — GPU-облако с оплатой по использованию. A100 стоит $0.49/час на spot-инстансах. Тренировка DeBERTa для InkCloak заняла 1 час.

**Когда ломается:** никогда, потому что я плачу только за использование. Нет ежемесячного платежа.

## GPU-инференс: NVIDIA P100 — $150 разово

Купил б/у NVIDIA P100 16GB за $150 на eBay. Стоит дома, подключена к десктопу. Запускает DeBERTa-инференс для InkCloak.

**Когда ломается:** P100 — это GPU 2017 года. Медленнее A100 в 5-10 раз. Для batch-инференса достаточно, для real-time с высокой нагрузкой — нет. Когда InkCloak вырастет, переключусь на RunPod Serverless.

## Домены: $15/мес

Единственная реальная стоимость. 5 доменов (.com, .app, .ai) через Cloudflare Registrar (дешевле всего). Примерно $15/мес в пересчёте на год.

## Когда бесплатное кончится

Реалистичный прогноз:

- **100 пользователей** — всё ещё $15/мес
- **1000 пользователей** — $30-50/мес (Vercel Pro + больше GPU)
- **10 000 пользователей** — $200-300/мес (Supabase Pro + RunPod Serverless)

К моменту 10K пользователей продукты должны генерировать достаточно revenue, чтобы покрыть инфраструктуру. Если нет — значит, продукт не нужен рынку, и инфраструктура — не самая большая проблема.

## Принцип

Не оптимизируй инфраструктуру до product-market fit. Используй бесплатные тарифы. Плати только когда growth вынуждает. Каждый доллар до PMF — это доллар, который мог пойти на итерацию продукта.
